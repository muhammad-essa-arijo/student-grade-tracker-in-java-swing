## Development Workflow

### For Initial Setup
1. Download all files to folder
2. Run `build.bat` (Windows) or `./build.sh` (Linux/Mac)

5. Archive as ZIP

## File Access Patterns

```
User starts application
    ↓
StudentGradeTrackerApp.java
    ├─ Creates new GradeBook (Model) - in memory
    ├─ Creates MainFrame (View) - displays UI
    └─ Creates StudentGradeController (Controller) - connects them
    
User interacts with UI
    ├─ Clicks buttons → Controller event handlers
    ├─ Model updated (in memory)
    └─ View refreshes from Model
    
User saves to CSV
    ├─ CSVService.saveToCSV()
    │   └─ Writes to disk
    └─ File created/updated
    
User loads from CSV
    ├─ CSVService.loadFromCSV()
    │   └─ Reads from disk
    └─ Model populated
    
Autosave timer (every 2 min)
    ├─ CSVService.saveToCSV()
    │   └─ Writes to configured file
    └─ Data persisted automatically
```
